# Implementation Plan: Bayesian Optimization for Sample Size Determination

## Overview

This document provides a comprehensive implementation plan for integrating Bayesian Optimization (BO) functionality for sample size determination into an R package. The goal is to find the minimum sample size `n` such that `power(n) >= target`, with asymmetric handling where undershooting the target power is heavily penalized.

**Two approaches are provided:**
1. **Penalty-based** (default): Single-objective BO with asymmetric penalty for constraint violations
2. **Constrained EIC**: True constrained BO using Expected Improvement with Constraints (EIC), leveraging surrogate uncertainty for probability of feasibility

---

## 1. Problem Specification

### 1.1 Mathematical Formulation

**Objective:** Find the minimum sample size `n` such that:
- `power(n) >= t` (target power constraint)
- Undershooting (`power(n) < t`) must be prevented at high cost
- Overshooting is acceptable but inefficient

**Key Characteristics:**
- Input: Sample size `n` (positive integer, typically range 10-10000)
- Output: Statistical power (continuous, bounded [0, 1])
- Relationship: Monotonically increasing (more samples → more power)
- Evaluation: Expensive (requires simulation)

### 1.2 Design Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Input transformation | `log(n)` | Power-sample size relationship is approximately linear on log scale; improves GP fit |
| Power transformation | `logit(power)` | Maps [0,1] to unbounded scale; homogeneous variance; better GP fit |
| Constraint handling | Penalty-based OR Constrained EIC | Penalty is simple/robust; EIC uses uncertainty for probabilistic feasibility |
| Surrogate model | Gaussian Process (default) or Random Forest | GP for smooth functions, RF for noisy/non-stationary |
| Acquisition function | EI (penalty) or custom EIC (constrained) | EIC = EI × P(feasible) for constrained approach |

### 1.3 Method Comparison

| Aspect | Penalty-Based | Constrained EIC |
|--------|---------------|-----------------|
| Complexity | Simple | More complex |
| Uncertainty use | Implicit via EI | Explicit P(feasible) from posterior |
| Conservatism control | Via penalty strength | Via κ parameter (confidence level) |
| Interpretability | Penalty weights | Probability of meeting target |
| Recommended for | General use | When explicit uncertainty quantification needed |

```r
#' Bayesian Optimization for Sample Size Determination
#'
#' Find the minimum sample size to achieve a target power level using

' Bayesian Optimization with asymmetric constraint handling.
#'
#' @param power_fun A function that takes a single argument `n` (sample size)
#'   and returns estimated power (numeric between 0 and 1). This function
#'   typically runs a simulation.
#' @param target Target power level to achieve (default: 0.80).
#' @param n_range Numeric vector of length 2 specifying the search range
#'   for sample size, e.g., `c(10, 500)`.
#' @param n_init Number of initial design points (default: 8).
#' @param n_iter Number of BO iterations after initial design (default: 20).
#' @param penalty_type Type of penalty for undershooting. One of:
#'   - `"exponential"` (default): Exponential penalty, severe
#'   - `"quadratic"`: Quadratic penalty, moderate
#'   - `"linear"`: Linear penalty, mild
#'   - `"step"`: Step function with large constant penalty
#' @param penalty_lambda Penalty strength parameter (default: 100).
#'   Higher values more strongly discourage undershooting.
#' @param surrogate Type of surrogate model: `"gp"` (Gaussian Process, default)
#'   or `"rf"` (Random Forest).
#' @param acq_function Acquisition function: `"ei"` (Expected Improvement, default),
#'   `"cb"` (Confidence Bound), or `"pi"` (Probability of Improvement).
#' @param overshoot_tolerance How much overshoot above target is acceptable
#'   before adding a small penalty (default: 0.05, i.e., 5 percentage points).
#' @param seed Random seed for reproducibility (default: NULL).
#' @param verbose Logical; print progress information (default: TRUE).
#' @param return_archive Logical; return full optimization archive (default: FALSE).
#'
#' @return A list of class `"bo_sample_size_result"` containing:
#'   \describe{
#'     \item{n_optimal}{Recommended sample size (integer)}
#'     \item{power_optimal}{Estimated power at optimal n}
#'     \item{n_evaluated}{Total number of function evaluations}
#'     \item{feasible}{Logical; whether a feasible solution was found}
#'     \item{all_results}{Data frame of all evaluated points}
#'     \item{archive}{Full bbotk archive (if return_archive = TRUE)}
#'     \item{surrogate}{Fitted surrogate model (if return_archive = TRUE)}
#'   }
#'
#' @export
#'
#' @examples
#' \dontrun
#' # Define a power simulation function
#' my_power_fun <- function(n) {
#'   # Example: two-sample t-test power
#'   power.t.test(n = n, delta = 0.5, sd = 1, type = "two.sample")$power
#' }
#'
#' # Find minimum sample size for 80% power
#' result <- bo_sample_size(
#'   power_fun = my_power_fun,
#'   target = 0.80,
#'   n_range = c(10, 200),
#'   n_iter = 15
#' )
#'
#' print(result)
#' plot(result)
#' }
bo_sample_size <- function(
    power_fun,
    target = 0.80,
    n_range = c(10, 1000),
    n_init = 8,
    n_iter = 20,
    penalty_type = c("exponential", "quadratic", "linear", "step"),
    penalty_lambda = 100,
    surrogate = c("gp", "rf"),
    acq_function = c("ei", "cb", "pi"),
    overshoot_tolerance = 0.05,
    seed = NULL,
    verbose = TRUE,
    return_archive = FALSE
) {

.
  # ===== Input Validation =====
  validate_bo_inputs(
    power_fun = power_fun,
    target = target,
    n_range = n_range,
    n_init = n_init,
    n_iter = n_iter,
    penalty_lambda = penalty_lambda,
    overshoot_tolerance = overshoot_tolerance
)

  penalty_type <- match.arg(penalty_type)
  surrogate <- match.arg(surrogate)
  acq_function <- match.arg(acq_function)

  if (!is.null(seed)) set.seed(seed)

  # ===== Setup =====
  if (verbose) message("Setting up Bayesian Optimization...")

  # Create penalty function
  penalty_fun <- create_penalty_function(
    type = penalty_type,
    lambda = penalty_lambda,
    target = target,
    overshoot_tolerance = overshoot_tolerance
  )

  # Create objective function
  objective <- create_bo_objective(
    power_fun = power_fun,
    penalty_fun = penalty_fun,
    target = target,
    n_range = n_range
  )

  # Create optimization instance
  instance <- create_opt_instance(
    objective = objective,
    n_range = n_range,
    n_init = n_init,
    n_iter = n_iter
  )

  # Configure surrogate
  surrogate_model <- create_surrogate(type = surrogate)

  # Configure acquisition function
  acq_fun <- create_acquisition_function(type = acq_function)

  # Configure acquisition optimizer
  acq_opt <- create_acq_optimizer()

  # ===== Run Optimization =====
  if (verbose) message("Running Bayesian Optimization...")

  optimizer <- mlr3mbo::opt("mbo",
    loop_function = mlr3mbo::bayesopt_ego,
    surrogate = surrogate_model,
    acq_function = acq_fun,
    acq_optimizer = acq_opt
  )

  # Suppress mlr3 logging if not verbose
  if (!verbose) {
    lgr::get_logger("mlr3")$set_threshold("warn")
    lgr::get_logger("bbotk")$set_threshold("warn")
  }

  optimizer$optimize(instance)

  # ===== Process Results =====
  if (verbose) message("Processing results...")

  result <- process_bo_results(
    instance = instance,
    target = target,
    n_range = n_range,
    return_archive = return_archive
  )

  if (verbose) {
    message(sprintf(
      "Optimization complete. Recommended n = %d (power = %.3f)",
      result$n_optimal, result$power_optimal
    ))
  }

  return(result)
}
```

### 4.2 Penalty Functions

**File:** `R/penalty_functions.R`

```r
#' Create Penalty Function for Constraint Violation
#'
#' @param type Penalty type
#' @param lambda Penalty strength
#' @param target Target power
#' @param overshoot_tolerance Acceptable overshoot
#'
#' @return A function that computes penalty given power value
#' @keywords internal
create_penalty_function <- function(type, lambda, target, overshoot_tolerance) {

  switch(type,
    "exponential" = function(power) {
      gap <- target - power
      if (gap > 0) {
        # Exponential penalty for undershooting
        # exp(k * gap) - 1 ensures penalty is 0 when gap = 0
        lambda * (exp(10 * gap) - 1)
      } else if (gap < -overshoot_tolerance) {
        # Small penalty for excessive overshoot (optional)
        0.1 * abs(gap - overshoot_tolerance)
      } else {
        0
      }
    },

    "quadratic" = function(power) {
      gap <- target - power
      if (gap > 0) {
        lambda * gap^2
      } else if (gap < -overshoot_tolerance) {
        0.1 * (gap + overshoot_tolerance)^2
      } else {
        0
      }
    },

    "linear" = function(power) {
      gap <- target - power
      if (gap > 0) {
        lambda * gap
      } else {
        0
      }
    },

    "step" = function(power) {
      gap <- target - power
      if (gap > 0) {
        lambda  # Constant large penalty
      } else {
        0
      }
    },

    stop("Unknown penalty type: ", type)
  )
}


#' Compute Asymmetric Loss
#'
#' Computes the combined objective value: sample size + penalty for constraint violation.
#'
#' @param n Sample size
#' @param power Estimated power
#' @param penalty_fun Penalty function
#'
#' @return Numeric objective value to minimize
#' @keywords internal
compute_asymmetric_loss <- function(n, power, penalty_fun) {
  n + penalty_fun(power)
}
```

```r
#' Logit Transformation
#'
#' Transform probability from (0,1) to (-Inf, Inf).
#'
#' @param p Probability value(s) in (0, 1)
#' @param eps Small value to avoid log(0) (default: 1e-8)
#'
#' @return Logit-transformed value(s)
#' @export
#'
#' @examples
#' logit(0.5)   # 0
#' logit(0.8)   # 1.386
#' logit(0.95)  # 2.944
logit <- function(p, eps = 1e-8) {
  p <- pmax(pmin(p, 1 - eps), eps)  # Avoid exact 0 or 1
  log(p / (1 - p))
}


#' Inverse Logit (Expit) Transformation
#'
#' Transform value from (-Inf, Inf) back to (0, 1).
#'
#' @param x Real value(s)
#'
#' @return Probability value(s) in (0, 1)
#' @export
#'
#' @examples
#' expit(0)      # 0.5
#' expit(1.386)  # ~0.8
#' expit(-2)     # ~0.12
expit <- function(x) {
  1 / (1 + exp(-x))
}


#' Transform Sample Size to Log Scale
#'
#' @param n Sample size (positive integer)
#' @return log(n)
#' @keywords internal
n_to_log <- function(n) {

  log(n)
}


#' Transform Log Scale Back to Sample Size
#'
#' @param log_n Log-transformed sample size
#' @return Rounded sample size (integer)
#' @keywords internal
log_to_n <- function(log_n) {
  round(exp(log_n))
}
```

### 4.4 Custom EIC Acquisition Function

**File:** `R/acq_function_eic.R`

This implements Expected Improvement with Constraints (EIC) for true constrained Bayesian optimization.

```r
#' Acquisition Function: Expected Improvement with Constraints (EIC)
#'
#' @description
#' Custom acquisition function that multiplies Expected Improvement by the

#' probability of feasibility:
#'
#' EIC(x) = EI(x) × P(power(x) >= target)
#'
#' The constraint (power) is modeled on the logit scale for better GP fit.
#' P(feasible) is computed from the GP posterior on the logit scale.
#'
#' @details
#' Works on:
#' - Log scale for sample size n: optimize log(n), transform back for evaluation
#' - Logit scale for power: model logit(power), compute P(feasible) on this scale
#'
#' The probability of feasibility is:
#' P(power >= target) = P(logit(power) >= logit(target))
#'                    = Phi((mu_logit - logit(target)) / sigma_logit)
#'
#' Optional conservatism via kappa parameter uses lower confidence bound:
#' effective_mean = mu_logit - kappa * sigma_logit
#'
#' @export
AcqFunctionEIC <- R6::R6Class(
  "AcqFunctionEIC",
  inherit = mlr3mbo::AcqFunction,

  public = list(
    #' @field surrogate_power Surrogate model for power (on logit scale)
    surrogate_power = NULL,

    #' @field target_power Target power level (probability scale, e.g., 0.80)
    target_power = NULL,

    #' @field target_logit Target power on logit scale
    target_logit = NULL,

    #' @field y_best Current best feasible objective value
    y_best = NULL,

    #' @field kappa Conservatism parameter for P(feasible)
    #'   0 = neutral (use posterior mean)
    #'   1.96 = conservative (97.5% LCB)
    #'   2.58 = very conservative (99.5% LCB)
    kappa = NULL,

    #' @description
    #' Initialize the EIC acquisition function
    #'
    #' @param surrogate Surrogate for objective (predicts log(n) or penalized objective)
    #' @param surrogate_power Surrogate for power (predicts logit(power))
    #' @param target_power Target power on probability scale (default 0.80)
    #' @param kappa Conservatism parameter. If > 0, uses
    #'   mean_logit - kappa*se instead of mean_logit for P(feasible).
    #'   Higher values = more conservative = higher confidence constraint is met.
    initialize = function(surrogate,
                          surrogate_power,
                          target_power = 0.80,
                          kappa = 0) {

      # Store constraint surrogate and target
      self$surrogate_power <- surrogate_power
      self$target_power <- target_power
      self$target_logit <- logit(target_power)
      self$kappa <- kappa

      # Initialize parent class
      super$initialize(
        id = "eic",
        constants = paradox::ps(),
        surrogate = surrogate,
        requires_predict_type_se = TRUE,
        direction = "minimize",
        label = "Expected Improvement with Constraints",
        man = NA_character_
      )
    },

    #' @description
    #' Update the acquisition function with current archive data
    #'
    #' @param archive The optimization archive
    update = function(archive) {
      super$update(archive)

      # Update constraint surrogate
      self$surrogate_power$update()

      # Find best feasible objective value
      # Archive should contain 'power' column from evaluations
      data <- archive$data

      if ("power" %in% names(data)) {
        feasible_mask <- data$power >= self$target_power
        if (any(feasible_mask)) {
          # Best = smallest objective value among feasible points
          self$y_best <- min(data$y[feasible_mask])
        } else {
          # No feasible points yet - use worst case to encourage exploration
          self$y_best <- max(data$y) + 1
        }
      } else {
        # Fallback if power not tracked
        self$y_best <- min(data$y)
      }

      invisible(self)
    },

    #' @description
    #' Evaluate the acquisition function at candidate points
    #'
    #' @param xdt data.table of candidate points (with log_n column)
    #' @return data.table with acq_eic column
    .eval_dt = function(xdt) {

      # ----- Predict objective (log_n) -----
      pred_obj <- self$surrogate$predict(xdt)
      mean_obj <- pred_obj$mean
      se_obj <- pred_obj$se

      # ----- Compute Expected Improvement -----
      # EI for minimization: E[max(y_best - Y, 0)]
      # where Y ~ N(mean, se^2)

      improvement <- self$y_best - mean_obj
      z <- improvement / se_obj

      ei <- improvement * pnorm(z) + se_obj * dnorm(z)
      ei[se_obj < 1e-8] <- 0  # No uncertainty = no expected improvement

      # ----- Predict constraint (logit scale) -----
      pred_power <- self$surrogate_power$predict(xdt)
      mean_logit <- pred_power$mean
      se_logit <- pred_power$se

      # ----- Compute Probability of Feasibility -----
      # P(power >= target) = P(logit(power) >= logit(target))
      # With GP: logit(power) ~ N(mean_logit, se_logit^2)
      # P(X >= c) = 1 - Phi((c - mu)/sigma) = Phi((mu - c)/sigma)

      # Optional conservatism: use lower confidence bound on logit
      if (self$kappa > 0) {
        effective_mean <- mean_logit - self$kappa * se_logit
      } else {
        effective_mean <- mean_logit
      }

      z_constraint <- (effective_mean - self$target_logit) / se_logit
      prob_feasible <- pnorm(z_constraint)

      # Handle edge cases
      prob_feasible[se_logit < 1e-8] <- as.numeric(
        mean_logit[se_logit < 1e-8] >= self$target_logit
      )

      # ----- EIC = EI × P(feasible) -----
      eic <- ei * prob_feasible

      # Return as data.table (mlr3mbo expects this format)
      data.table::data.table(acq_eic = eic)
    }
  )
)
```

### 4.5 Constrained BO Main Function

**File:** `R/bo_sample_size_eic.R`

```r
#' Bayesian Optimization for Sample Size with Constrained EIC
#'
#' Find the minimum sample size using true constrained Bayesian Optimization
#' with Expected Improvement with Constraints (EIC). This method explicitly
#' models the power constraint and uses surrogate uncertainty to compute
#' probability of feasibility.
#'
#' @param power_fun A function that takes a single argument `n` (sample size)
#'   and returns estimated power (numeric between 0 and 1).
#' @param target Target power level to achieve (default: 0.80).
#' @param n_range Numeric vector of length 2 specifying the search range
#'   for sample size, e.g., `c(10, 500)`.
#' @param n_init Number of initial design points (default: 8).
#' @param n_iter Number of BO iterations after initial design (default: 20).
#' @param kappa Conservatism parameter for probability of feasibility:
#'   - `0`: Use posterior mean (neutral)
#'   - `1.96`: Use 97.5% lower confidence bound (conservative)
#'   - `2.58`: Use 99.5% LCB (very conservative)
#'   Higher values give more confidence that the constraint is satisfied.
#' @param surrogate Type of surrogate model: `"gp"` (Gaussian Process, default)
#'   or `"rf"` (Random Forest).
#' @param seed Random seed for reproducibility (default: NULL).
#' @param verbose Logical; print progress information (default: TRUE).
#'
#' @return A list of class `"bo_sample_size_eic_result"` containing:
#'   \describe{
#'     \item{n_optimal}{Recommended sample size from evaluated points}
#'     \item{power_optimal}{Achieved power at optimal n}
#'     \item{feasible}{Whether a feasible solution was found}
#'     \item{n_surrogate}{Surrogate-based recommendation}
#'     \item{power_surrogate}{Predicted power at surrogate recommendation}
#'     \item{power_surrogate_lcb}{Lower confidence bound on power (if kappa > 0)}
#'     \item{all_results}{Data frame of all evaluated points}
#'     \item{surrogate_obj}{Fitted objective surrogate}
#'     \item{surrogate_power}{Fitted power surrogate}
#'     \item{surrogate_grid}{Grid of surrogate predictions for plotting}
#'   }
#'
#' @details
#' This method uses two separate surrogate models:
#' 1. Surrogate for objective (log(n)): predicts the objective value
#' 2. Surrogate for constraint (logit(power)): predicts power on unbounded scale
#'
#' The acquisition function is EIC = EI × P(power >= target), where
#' P(feasible) is computed from the GP posterior on the logit scale.
#'
#' The kappa parameter controls conservatism:
#' - kappa = 0: Recommend based on posterior mean (may undershoot target)
#' - kappa = 1.96: Recommend based on 97.5% lower confidence bound
#' - kappa = 2.58: Recommend based on 99.5% LCB (very conservative)
#'
#' @export
#'
#' @examples
#' \dontrun{
#' # Define power function
#' power_ttest <- function(n) {
#'   power.t.test(n = n, delta = 0.5, sd = 1, type = "two.sample")$power
#' }
#'
#' # Run constrained BO with conservative settings
#' result <- bo_sample_size_eic(
#'   power_fun = power_ttest,
#'   target = 0.80,
#'   n_range = c(20, 200),
#'   n_iter = 15,
#'   kappa = 1.96,  # 97.5% confidence
#'   seed = 42
#' )
#'
#' print(result)
#' plot(result)
#' }
bo_sample_size_eic <- function(
    power_fun,
    target = 0.80,
    n_range = c(10, 1000),
    n_init = 8,
    n_iter = 20,
    kappa = 0,
    surrogate = c("gp", "rf"),
    seed = NULL,
    verbose = TRUE
) {

  # ===== Input Validation =====
  validate_bo_inputs_eic(
    power_fun = power_fun,
    target = target,
    n_range = n_range,
    n_init = n_init,
    n_iter = n_iter,
    kappa = kappa
  )

  surrogate <- match.arg(surrogate)
  if (!is.null(seed)) set.seed(seed)

  # ===== Setup =====
  if (verbose) message("Setting up Constrained Bayesian Optimization (EIC)...")

  # Define search space (log scale for n)
  domain <- paradox::ps(
    log_n = paradox::p_dbl(lower = log(n_range[1]), upper = log(n_range[2]))
  )

  # Define codomain (what objective returns)
  codomain <- paradox::ps(
    y = paradox::p_dbl(tags = "minimize"),
    power_logit = paradox::p_dbl(),
    n_actual = paradox::p_dbl(),
    power = paradox::p_dbl(),
    feasible = paradox::p_dbl(),
    gap = paradox::p_dbl()
  )

  # Create objective function (returns both objective and constraint)
  obj_fun <- create_eic_objective(power_fun, target)

  objective <- bbotk::ObjectiveRFun$new(
    fun = obj_fun,
    domain = domain,
    codomain = codomain,
    properties = "deterministic"
  )

  # Create optimization instance
  instance <- bbotk::OptimInstanceBatchSingleCrit$new(
    objective = objective,
    terminator = bbotk::trm("evals", n_evals = n_init + n_iter)
  )

  # Generate and evaluate initial design
  design <- paradox::generate_design_sobol(domain, n = n_init)$data
  instance$eval_batch(design)

  # ===== Create Surrogates =====
  if (verbose) message("Configuring surrogates...")

  # Surrogate for objective (log_n)
  surrogate_obj <- create_surrogate_for_eic(surrogate, instance$archive, "y")

  # Surrogate for constraint (power_logit)
  surrogate_power <- create_surrogate_for_eic(surrogate, instance$archive, "power_logit")

  # Create EIC acquisition function
  acq_function <- AcqFunctionEIC$new(
    surrogate = surrogate_obj,
    surrogate_power = surrogate_power,
    target_power = target,
    kappa = kappa
  )

  # Create acquisition optimizer
  acq_optimizer <- mlr3mbo::acqo(
    optimizer = bbotk::opt("random_search", batch_size = 1000),
    terminator = bbotk::trm("evals", n_evals = 1000)
  )

  # ===== Run BO Loop =====
  if (verbose) message("Running optimization...")

  for (i in seq_len(n_iter)) {

    # Update surrogates
    surrogate_obj$update()
    surrogate_power$update()

    # Update acquisition function
    acq_function$update(instance$archive)

    # Optimize acquisition function
    best_candidate <- optimize_acq_function_eic(
      acq_function, domain, acq_optimizer
    )

    # Evaluate on true objective
    instance$eval_batch(data.table::as.data.table(best_candidate))

    if (verbose && i %% 5 == 0) {
      best_n <- get_best_feasible_n(instance$archive, target)
      message(sprintf("Iteration %d/%d: best feasible n = %s",
                      i, n_iter, ifelse(is.na(best_n), "none", best_n)))
    }
  }

  # ===== Process Results =====
  if (verbose) message("Processing results...")

  result <- extract_eic_results(
    archive = instance$archive,
    target = target,
    kappa = kappa,
    surrogate_obj = surrogate_obj,
    surrogate_power = surrogate_power
  )

  if (verbose) {
    if (result$feasible) {
      message(sprintf(
        "Optimization complete. Recommended n = %d (power = %.3f)",
        result$n_optimal, result$power_optimal
      ))
    } else {
      message("Optimization complete. No feasible solution found in evaluated points.")
      if (!is.na(result$n_surrogate)) {
        message(sprintf("Surrogate suggests n = %d (predicted power = %.3f)",
                        result$n_surrogate, result$power_surrogate))
      }
    }
  }

  return(result)
}


#' Create Objective Function for EIC
#'
#' @param power_fun User's power function
#' @param target Target power
#' @return Function for bbotk
#' @keywords internal
create_eic_objective <- function(power_fun, target) {

  force(power_fun)
  force(target)

  function(xs) {
    # Back-transform from log scale
    log_n <- xs$log_n
    n <- round(exp(log_n))

    # Evaluate power
    power <- power_fun(n)

    # Transform power to logit scale for surrogate
    power_logit <- logit(power)

    # Return multiple outputs
    list(
      y = log_n,                              # Objective: minimize log(n)
      power_logit = power_logit,              # Constraint on logit scale
      n_actual = n,                           # For inspection
      power = power,                          # For inspection
      feasible = as.numeric(power >= target), # For inspection
      gap = target - power                    # For inspection
    )
  }
}


#' Create Surrogate for EIC
#'
#' @param type Surrogate type ("gp" or "rf")
#' @param archive bbotk archive
#' @param y_col Column to predict
#' @return SurrogateLearner object
#' @keywords internal
create_surrogate_for_eic <- function(type, archive, y_col) {

  if (type == "gp") {
    learner <- mlr3::lrn("regr.km",
      covtype = "matern3_2",
      optim.method = "gen",
      control = list(trace = FALSE),
      nugget.estim = FALSE,
      nugget = 1e-8
    )
  } else {
    learner <- mlr3::lrn("regr.ranger",
      num.trees = 500,
      keep.inbag = TRUE,
      se.method = "jack"
    )
  }

  surrogate <- mlr3mbo::srlrn(learner, archive = archive)
  surrogate$cols_y <- y_col

  return(surrogate)
}


#' Optimize EIC Acquisition Function
#'
#' @param acq_function AcqFunctionEIC object
#' @param domain Search space
#' @param acq_optimizer Acquisition optimizer
#' @return Best candidate point
#' @keywords internal
optimize_acq_function_eic <- function(acq_function, domain, acq_optimizer) {

  # Create temporary instance for acquisition optimization
  acq_codomain <- paradox::ps(acq_eic = paradox::p_dbl(tags = "maximize"))

  acq_objective <- bbotk::ObjectiveRFun$new(
    fun = function(xs) {
      xdt <- data.table::as.data.table(xs)
      as.list(acq_function$.eval_dt(xdt))
    },
    domain = domain$clone(),
    codomain = acq_codomain
  )

  acq_instance <- bbotk::OptimInstanceBatchSingleCrit$new(
    objective = acq_objective,
    terminator = bbotk::trm("evals", n_evals = 1000)
  )

  # Optimize
  acq_optimizer$optimize(acq_instance)

  # Return best candidate
  acq_instance$result_x_domain
}

#' Extract Results from EIC Optimization
#'
#' @param archive bbotk archive
#' @param target Target power
#' @param kappa Conservatism parameter
#' @param surrogate_obj Objective surrogate
#' @param surrogate_power Power surrogate
#' @return List of results
#' @keywords internal
extract_eic_results <- function(archive, target, kappa, surrogate_obj, surrogate_power) {

  data <- archive$data

  # ----- From evaluated points -----
  feasible <- data[power >= target]

  if (nrow(feasible) > 0) {
    best_idx <- which.min(feasible$n_actual)
    n_optimal <- feasible$n_actual[best_idx]
    power_optimal <- feasible$power[best_idx]
    is_feasible <- TRUE
  } else {
    n_optimal <- NA
    power_optimal <- NA
    is_feasible <- FALSE
  }

  # ----- From surrogate (with optional conservatism) -----
  log_n_grid <- seq(min(data$log_n), max(data$log_n), length.out = 500)
  grid_dt <- data.table::data.table(log_n = log_n_grid)

  # Update and predict
  surrogate_obj$update()
  surrogate_power$update()

  pred_power <- surrogate_power$predict(grid_dt)
  target_logit <- logit(target)

  # Apply conservatism
  if (kappa > 0) {
    effective_logit <- pred_power$mean - kappa * pred_power$se
  } else {
    effective_logit <- pred_power$mean
  }

  # Build grid with predictions
  grid_dt[, `:=`(
    pred_logit = pred_power$mean,
    pred_se = pred_power$se,
    effective_logit = effective_logit,
    pred_power = expit(pred_power$mean),
    pred_power_lcb = expit(effective_logit),
    n = round(exp(log_n))
  )]

  # Find smallest n where effective power meets target
  predicted_feasible <- grid_dt[effective_logit >= target_logit]

  if (nrow(predicted_feasible) > 0) {
    best_pred_idx <- which.min(predicted_feasible$log_n)
    n_surrogate <- predicted_feasible$n[best_pred_idx]
    power_surrogate <- predicted_feasible$pred_power[best_pred_idx]
    power_surrogate_lcb <- predicted_feasible$pred_power_lcb[best_pred_idx]
  } else {
    n_surrogate <- NA
    power_surrogate <- NA
    power_surrogate_lcb <- NA
  }

  # Build result
  result <- list(
    # Primary recommendation (from evaluated points)
    n_optimal = n_optimal,
    power_optimal = power_optimal,
    feasible = is_feasible,

    # Surrogate-based recommendation
    n_surrogate = n_surrogate,
    power_surrogate = power_surrogate,
    power_surrogate_lcb = power_surrogate_lcb,

    # Settings
    target_power = target,
    kappa = kappa,

    # Full data
    all_results = data,
    surrogate_grid = grid_dt,
    surrogate_obj = surrogate_obj,
    surrogate_power = surrogate_power
  )

  class(result) <- c("bo_sample_size_eic_result", "list")
  return(result)
}
```

### 4.7 Objective Function Construction (Penalty-Based)

**File:** `R/objective_functions.R`

```r
#' Create BO Objective Function
#'
#' Wraps the user's power function and penalty into a bbotk-compatible objective.
#'
#' @param power_fun User's power simulation function
#' @param penalty_fun Penalty function for constraint violations
#' @param target Target power level
#' @param n_range Sample size range
#'
#' @return ObjectiveRFun object
#' @keywords internal
create_bo_objective <- function(power_fun, penalty_fun, target, n_range) {

  # Objective function operating on log scale
  obj_fun <- function(xs) {
    # Back-transform from log scale
    n <- round(exp(xs$log_n))

    # Clamp to valid range
    n <- max(n_range[1], min(n_range[2], n))

    # Evaluate power (this is the expensive operation)
    power <- power_fun(n)

    # Validate power output
    if (!is.numeric(power) || length(power) != 1 || is.na(power)) {
      stop("power_fun must return a single numeric value")
    }
    if (power < 0 || power > 1) {
      warning("power_fun returned value outside [0, 1]: ", power)
      power <- max(0, min(1, power))
    }

    # Compute objective (to minimize)
    y <- compute_asymmetric_loss(n, power, penalty_fun)

    # Return all values for monitoring
    list(
      y = y,
      power = power,
      n_actual = n,
      gap = target - power,
      feasible = as.numeric(power >= target)
    )
  }

  # Define parameter space (log-transformed)
  domain <- paradox::ps(
    log_n = paradox::p_dbl(
      lower = log(n_range[1]),
      upper = log(n_range[2])
    )
  )

  # Define codomain (output space)
  codomain <- paradox::ps(
    y = paradox::p_dbl(tags = "minimize"),
    power = paradox::p_dbl(),
    n_actual = paradox::p_dbl(),
    gap = paradox::p_dbl(),
    feasible = paradox::p_dbl()
  )

  # Create objective
  bbotk::ObjectiveRFun$new(
    fun = obj_fun,
    domain = domain,
    codomain = codomain,
    id = "sample_size_power"
  )
}


#' Create Optimization Instance
#'
#' @param objective ObjectiveRFun object
#' @param n_range Sample size range
#' @param n_init Number of initial points
#' @param n_iter Number of BO iterations
#'
#' @return OptimInstanceBatchSingleCrit object
#' @keywords internal
create_opt_instance <- function(objective, n_range, n_init, n_iter) {

  # Total evaluations = initial design + BO iterations
  n_evals <- n_init + n_iter

  instance <- bbotk::OptimInstanceBatchSingleCrit$new(
    objective = objective,
    terminator = bbotk::trm("evals", n_evals = n_evals)
  )

  # Generate initial design using Sobol sequence on log scale
  design <- paradox::generate_design_sobol(
    param_set = objective$domain,
    n = n_init
  )$data

  # Evaluate initial design
  instance$eval_batch(design)

  return(instance)
}
```

```r
#' Process BO Results
#'
#' Extract and format results from the optimization.
#'
#' @param instance OptimInstanceBatchSingleCrit
#' @param target Target power
#' @param n_range Sample size range
#' @param return_archive Whether to include full archive
#'
#' @return List of class "bo_sample_size_result"
#' @keywords internal
process_bo_results <- function(instance, target, n_range, return_archive) {

  # Extract archive data
  archive_data <- instance$archive$data

  # Convert log_n back to n
  archive_data[, n := round(exp(log_n))]

  # Identify feasible points
  feasible_data <- archive_data[power >= target]

  if (nrow(feasible_data) > 0) {
    # Find minimum n among feasible points
    best_idx <- which.min(feasible_data$n_actual)
    n_optimal <- feasible_data$n_actual[best_idx]
    power_optimal <- feasible_data$power[best_idx]
    feasible <- TRUE
  } else {
    # No feasible point found - return point closest to target
    warning("No feasible solution found. Returning point with highest power.")
    best_idx <- which.max(archive_data$power)
    n_optimal <- archive_data$n_actual[best_idx]
    power_optimal <- archive_data$power[best_idx]
    feasible <- FALSE
  }

  # Prepare all results data frame
  all_results <- data.frame(
    iteration = seq_len(nrow(archive_data)),
    n = archive_data$n_actual,
    power = archive_data$power,
    objective = archive_data$y,
    feasible = archive_data$feasible == 1
  )

  # Create result object
  result <- list(
    n_optimal = as.integer(n_optimal),
    power_optimal = power_optimal,
    target = target,
    n_evaluated = nrow(archive_data),
    feasible = feasible,
    all_results = all_results,
    n_range = n_range
  )

  if (return_archive) {
    result$archive <- instance$archive
    result$instance <- instance
  }

  class(result) <- "bo_sample_size_result"
  return(result)
}
```

**File:** `R/validators.R`

```r
#' Validate BO Inputs
#'
#' @param power_fun Power function
#' @param target Target power
#' @param n_range Sample size range
#' @param n_init Initial design size
#' @param n_iter Number of iterations
#' @param penalty_lambda Penalty parameter
#' @param overshoot_tolerance Overshoot tolerance
#'
#' @keywords internal
validate_bo_inputs <- function(power_fun, target, n_range, n_init, n_iter,
                                penalty_lambda, overshoot_tolerance) {

  # Check power_fun is a function
  checkmate::assert_function(power_fun, args = "n")

  # Test power_fun with a sample value
  test_n <- round(mean(n_range))
  test_result <- tryCatch(
    power_fun(test_n),
    error = function(e) {
      stop(sprintf(
        "power_fun failed with test input n=%d: %s",
        test_n, e$message
      ))
    }
  )

  if (!is.numeric(test_result) || length(test_result) != 1) {
    stop("power_fun must return a single numeric value")
  }

  # Check target
  checkmate::assert_number(target, lower = 0, upper = 1)
  if (target < 0.5 || target > 0.99) {
    warning("Unusual target power: ", target, ". Typical values are 0.80 or 0.90.")
  }

  # Check n_range
  checkmate::assert_numeric(n_range, len = 2, lower = 1, any.missing = FALSE)
  if (n_range[1] >= n_range[2]) {
    stop("n_range[1] must be less than n_range[2]")
  }
  if (n_range[1] < 2) {
    stop("Minimum sample size must be at least 2")
  }

  # Check n_init and n_iter
  checkmate::assert_count(n_init, positive = TRUE)
  checkmate::assert_count(n_iter, positive = TRUE)

  if (n_init < 4) {
    warning("n_init < 4 may lead to poor surrogate fit. Consider increasing.")
  }

  # Check penalty_lambda
  checkmate::assert_number(penalty_lambda, lower = 0)
  if (penalty_lambda < 1) {
    warning("penalty_lambda < 1 provides weak constraint enforcement.")
  }

  # Check overshoot_tolerance
  checkmate::assert_number(overshoot_tolerance, lower = 0, upper = 0.5)

  invisible(TRUE)
}


#' Validate Power Function More Thoroughly
#'
#' Performs additional validation of the power function behavior.
#'
#' @param power_fun Power function to validate
#' @param n_range Sample size range
#' @param n_test Number of test points
#'
#' @return Logical; TRUE if validation passes
#' @export
validate_power_function <- function(power_fun, n_range, n_test = 5) {

  test_points <- round(seq(n_range[1], n_range[2], length.out = n_test))
  results <- numeric(n_test)

  for (i in seq_along(test_points)) {
    results[i] <- power_fun(test_points[i])
  }

  # Check results are valid
  if (any(is.na(results))) {
    warning("power_fun returned NA for some inputs")
    return(FALSE)
  }

  if (any(results < 0) || any(results > 1)) {
    warning("power_fun returned values outside [0, 1]")
    return(FALSE)
  }

  # Check monotonicity (power should generally increase with n)
  if (!all(diff(results) >= -0.1)) {  # Allow small decreases due to simulation noise
    warning("power_fun may not be monotonically increasing in n")
  }

  # Check range coverage
  if (max(results) < 0.5) {
    warning("Maximum observed power is low. Consider expanding n_range upper bound.")
  }

  if (min(results) > 0.9) {
    warning("Minimum observed power is high. Consider reducing n_range lower bound.")
  }

  return(TRUE)
}
```

```r
#' Recommend Sample Size with Safety Margin
#'
#' Returns a recommended sample size with a safety margin to account for
#' estimation uncertainty.
#'
#' @param result bo_sample_size_result object
#' @param margin Multiplicative safety margin (default: 1.1 = 10% increase)
#'
#' @return Integer recommended sample size
#' @export
recommend_with_margin <- function(result, margin = 1.1) {

  if (!result$feasible) {
    warning("No feasible solution found")
    return(NA_integer_)
  }

  n_margin <- ceiling(result$n_optimal * margin)

  # Ensure within original range
  n_margin <- min(n_margin, result$n_range[2])

  return(as.integer(n_margin))
}
```

## Introduction

This vignette demonstrates the constrained Bayesian Optimization approach
using Expected Improvement with Constraints (EIC). This method explicitly
models the power constraint and uses surrogate uncertainty to compute
the probability of feasibility.

## When to Use Constrained BO

Use `bo_sample_size_eic()` when:

- You want explicit uncertainty quantification
- You need probabilistic guarantees on meeting the target
- The penalty-based approach is too aggressive/conservative

## Basic Usage

```r

  power.t.test(n = n, delta = 0.5, sd = 1, type = "two.sample")$power
```

## Understanding the Conservatism Parameter (kappa)

The `kappa` parameter controls how conservative the optimization is:

- **kappa = 0**: Uses posterior mean for P(feasible). May recommend
  sample sizes that have ~50% chance of meeting target.

- **kappa = 1.96**: Uses 97.5% lower confidence bound. Recommendations
  have high probability (97.5%) of meeting the target.

- **kappa = 2.58**: Uses 99.5% LCB. Very conservative.

## How It Works

### Transformations

1. **Sample size**: Optimized on log scale (`log(n)`) for better GP fit
2. **Power**: Modeled on logit scale (`logit(power)`) for unbounded support

### Acquisition Function: EIC

The Expected Improvement with Constraints is:

$$EIC(x) = EI(x) \times P(power(x) \geq target)$$

Where:
- $EI(x)$ is standard Expected Improvement for minimizing sample size
- $P(feasible)$ is computed from the GP posterior on the logit scale

### Probability of Feasibility

With a GP surrogate for `logit(power)`:

$$P(power \geq target) = \Phi\left(\frac{\mu_{logit} - logit(target)}{\sigma_{logit}}\right)$$

With conservatism (kappa > 0):

$$P(feasible) = \Phi\left(\frac{(\mu_{logit} - \kappa \cdot \sigma_{logit}) - logit(target)}{\sigma_{logit}}\right)$$

## Interpreting Results

The result object contains:

- `n_optimal`: Best n from evaluated points that meets the target
- `power_optimal`: Achieved power at that n
- `n_surrogate`: Surrogate-based recommendation (uses kappa if > 0)
- `power_surrogate`: Predicted power at surrogate recommendation
- `power_surrogate_lcb`: Lower confidence bound on power (if kappa > 0)

## 9. Notes for Implementation

1. **Log transformation**: The sample size is optimized on the log scale internally. This improves GP performance since the power-n relationship is more linear on log scale.

2. **Logit transformation** (EIC only): Power is modeled on the logit scale for unbounded support and more homogeneous variance structure.

3. **Penalty tuning**: The default `penalty_lambda = 100` should work for most cases. Increase if the optimizer keeps finding infeasible solutions.

4. **Kappa selection**: Start with kappa = 0 for exploration, then increase to 1.96 or 2 for production recommendations.

5. **Initial design**: Uses Sobol sequences for space-filling. Consider Latin Hypercube as alternative.

6. **Noisy evaluations**: If power is estimated via simulation with high variance, consider:
   - Increasing `nsim` in the power function
   - Using Random Forest surrogate (more robust to noise)
   - Adding nugget term to GP
   - Using higher kappa for more conservative recommendations

7. **Integer constraint**: Sample size must be integer, but we optimize on continuous log scale and round. This works well in practice.

8. **Monotonicity**: The power function should be monotonically increasing. The current implementation doesn't enforce this but warns if violated.

9. **Two-surrogate architecture** (EIC): The EIC approach maintains separate surrogates for objective (log_n) and constraint (logit_power). Both are updated after each evaluation.

10. **Probability of feasibility**: Computing P(power >= target) from the GP posterior provides a principled way to handle constraint uncertainty, unlike penalty-based approaches which implicitly encode uncertainty through the penalty weight.
